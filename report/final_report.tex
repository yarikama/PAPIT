\begin{abstract}
Multimodal Large Language Models (MLLMs) such as LLaVA encode images
as hundreds of visual patch tokens, most of which are irrelevant to the
user's query.
We propose \textbf{PAPIT} (\textbf{P}rompt-\textbf{A}ware \textbf{P}runing
for \textbf{I}mage \textbf{T}okens), a training-free token pruning framework
that scores visual patch tokens against the user's text prompt using a
hybrid of GradCAM and MaskCLIP value features, retaining only the
top-$k$ most relevant tokens before LLaVA inference.
Efficiency experiments confirm that pruning to 25\% of tokens reduces
attention FLOPs by 90\%.
End-to-end latency benchmarks on LLaVA-1.5-7B (4-bit NF4, 100 images,
5 timed runs) reveal that original PAPIT scoring adds $\approx$130\,ms
fixed overhead, limiting speedup to T4 only (\textbf{1.36$\times$} at
$k{=}0.25$, 646\,ms vs.\ 876\,ms).
We address this with \emph{value\_free} scoring, which captures
\texttt{v\_proj} vectors during the existing ViT forward pass at
$\approx$18\,ms overhead, achieving \textbf{1.65$\times$} on T4 and
\textbf{1.34$\times$} on NVIDIA L4 (baseline 445\,ms), where original
PAPIT fails at every retention level (best 0.98$\times$).
Both GPU tiers are eligible under deployment guardrails with value\_free.
Accuracy evaluation across all three benchmarks (300 samples each) shows
PAPIT consistently approaches or surpasses random pruning: on GQA it
surpasses random at $k{=}75\%$ (70.0\% vs 69.3\%); on VQA~v2 it
matches random within 0.1 points at $k{=}75\%$ (73.3\% vs 73.4\%);
on TextVQA, PAPIT's text-region patch recall \emph{exceeds} random at
$k{=}25\%$ (40.8\% vs 33.9\%), directly evidencing prompt-aware selection.
\end{abstract}

\section{Introduction}

LLaVA-1.5 processes a 336$\times$336 image as a $24\!\times\!24 = 576$
patch token grid via a CLIP ViT-L/14 vision encoder.
Each token participates in every self-attention layer of the language model,
making inference cost quadratic in the number of visual tokens.
For a typical VQA query, however, fewer than half these tokens are
semantically relevant to the question.

\textbf{Global (random) pruning} discards tokens uniformly at a fixed ratio,
blind to the query content.
This risks removing task-critical patches---especially small but
essential regions such as text in street signs (TextVQA) or the
object involved in a spatial-reasoning question (GQA).

\textbf{PAPIT} instead uses the user's prompt to \emph{score} every patch
before selecting which ones to keep.
A naïve approach---cosine similarity between CLIP's output patch embeddings
and the text embedding---performs \emph{below random} (50.3\% vs.\ 64.3\%
on GQA at $k{=}25\%$), because CLIP's contrastive objective aligns only
the global CLS token with text, not individual patch tokens.
We instead exploit the internal structure of the transformer: in the final
ViT attention layer, the CLS output is a weighted sum over \emph{value}
projections, so each value vector directly encodes that patch's contribution
to the global image representation---making it comparable to the text
embedding without any extra forward pass.

% ============================================================
\section{Related Work}

Token pruning for Vision Transformers has been studied for image
classification~\cite{rao2021dynamicvit, meng2022adavit}, where a learned
predictor module decides which tokens to drop.
In the multimodal setting, FastV~\cite{chen2024fastv} prunes visual tokens
based on attention scores inside the LLM, requiring a forward pass before
pruning.
PAPIT differs in that pruning happens \emph{before} the LLM sees the image,
using a lightweight cross-modal scoring step ($<$30\,ms) that is entirely
query-conditioned and requires no training.

\subsection{Cross-Modal Scoring: Hybrid GradCAM / Value Features}

Naïve patch--text cosine similarity in CLIP's output space performs
\emph{below random} because CLIP's contrastive objective aligns only the
CLS (global) token with text; individual patch tokens are not explicitly
aligned (see Table~\ref{tab:ablation}, row 1: 50.3\% at $k{=}25\%$ vs.\
random 64.3\%).  We evaluate two principled alternatives grounded in
how transformers form their CLS representation.

\paragraph{GradCAM scoring.}
Let $H^{(L-1)} = \{h_1,\ldots,h_n\}$ denote patch activations at the
penultimate CLIP ViT encoder layer, and $\text{sim}$ the cosine similarity
between the CLIP CLS image embedding and the text embedding $T$.
We attribute $\text{sim}$ to each patch via GradCAM~\cite{selvaraju2017gradcam}:
\begin{equation}
  s^{\text{gc}}_i = \operatorname{ReLU}\!\left(
        \sum_{d}\, \frac{\partial\,\text{sim}}{\partial\, h_{id}}\cdot h_{id}
        \right).
  \label{eq:gradcam}
\end{equation}
Backpropagating through the final attention layer captures each patch's
\emph{causal} contribution to the global text-image alignment.

\paragraph{Value-feature scoring (MaskCLIP).}
Let $V = \{v_1,\ldots,v_n\}$ be the value vectors from the last CLIP
attention layer (CLS excluded).  In standard self-attention, the CLS
output is $\sum_j \alpha_{0j}\,W_V v_j$: a weighted sum of projected value
vectors.  Therefore $W_V v_i$ encodes exactly what patch~$i$ contributes
to the global CLS representation---it is already in the visual output
space and directly comparable to the text embedding~$T$
\cite{zhou2022maskclip}:
\begin{equation}
  s^{\text{val}}_i =
  \frac{\langle\, \mathbf{W}_v\,v_i,\; T \rangle}
       {\|\mathbf{W}_v\,v_i\|\,\|T\|},
  \label{eq:value}
\end{equation}
where $\mathbf{W}_v$ is the CLIP visual projection.
No backpropagation is required; $v_i$ is captured for free via a hook
during the already-running ViT forward pass ($\approx$18\,ms overhead
for text encoding only).

\paragraph{Hybrid routing.}
On GQA, GradCAM outperforms value features at aggressive retention
($k \leq 25\%$), while value features match or beat random pruning at
$k \geq 50\%$ (Table~\ref{tab:ablation}).
We therefore use GradCAM when $k \leq 37.5\%$ and value features otherwise.
The top-$k$ patches by the selected score are retained:
$V_{\text{pruned}} = \{h_i \mid s_i \in \text{TopK}(\{s_j\}_{j=1}^{n},\, k)\}$.

\subsection{Spatial Anchoring}

After top-$k$ selection, the pruned tokens preserve their original 2-D
grid indices as position IDs before being passed to the MLP projector.
One \emph{anchor token} (mean of all dropped tokens) is appended to
provide global context, giving a final sequence of length $k+1$.

\subsection{Efficiency Reduction}

Self-attention cost scales quadratically with sequence length.
Let $L_{\text{text}}$ be the number of text tokens (typically $\approx$50).
Relative FLOPs compared to unpruned inference is:
\begin{equation}
  \text{FLOPs}_{\text{rel}} =
  \frac{(k + L_{\text{text}})^2}{(n + L_{\text{text}})^2}.
  \label{eq:flops}
\end{equation}
For LLaVA-1.5 with $n=576$: retaining $k=25\%$ ($144$ tokens) yields
$\text{FLOPs}_{\text{rel}} \approx 0.10$, a $\sim$10$\times$ reduction.

\subsection{Implementation}

PAPIT operates at the token level between the CLIP ViT encoder and the
LLaVA MLP projector, requiring no modification to LLM weights.
We use \texttt{clip-vit-large-patch14} (matching LLaVA-1.5's vision tower)
via HuggingFace Transformers.
The full codebase is modular: \texttt{papit.core} (scorer),
\texttt{papit.integration} (LLaVA hook),
\texttt{papit.benchmark} (evaluation runner).

\noindent\textbf{Pipeline component breakdown.}
On T4, the ViT encoder accounts for $\approx$105\,ms (12\%) of the
876\,ms unpruned baseline; the LLM+MLP projector accounts for
$\approx$771\,ms (88\%).
CLIP text encoding adds $\approx$18\,ms; however, the full PAPIT scoring
step (visual feature projection $+$ cosine similarities over 576 patches)
adds $\approx$130\,ms of wall-clock overhead, visible as the
PAPIT$-$Random gap at each $k$.
Since PAPIT prunes \emph{after} the ViT, the ViT cost is fixed
regardless of $k$; only the dominant LLM+MLP portion scales with
retained tokens.
The theoretical speedup ceiling at $k{=}0.25$ on T4 is
$\tfrac{876}{105+130+0.25\times771}=2.05\times$ for original PAPIT
and $\tfrac{876}{105+18+0.25\times771}=2.78\times$ for value\_free scoring.

Key observations:
(1) \textbf{T4 — value\_free achieves highest speedup; original PAPIT
also eligible.}
value\_free achieves \textbf{1.65$\times$} at $k{=}0.25$ (531\,ms
vs.\ 876\,ms, saving 345\,ms per image) and \textbf{1.33$\times$} at
$k{=}0.50$ (657\,ms vs.\ 876\,ms); all three retention levels are eligible.
Original PAPIT achieves 1.36$\times$ at $k{=}0.25$ (646\,ms) and
1.10$\times$ at $k{=}0.50$ (796\,ms), also meeting deployment guardrails.
(2) \textbf{L4 — value\_free recovers speedup; original PAPIT fails.}
With the faster L4 (unpruned 445\,ms), original PAPIT is \emph{slower}
than unpruned at every $k$ (0.82--0.98$\times$): the $\approx$120\,ms
CLIP projection overhead exceeds all LLM token savings.
value\_free (overhead $\approx$18\,ms) achieves \textbf{1.34$\times$}
at $k{=}0.25$ (332\,ms vs.\ 445\,ms) and 1.13$\times$ at $k{=}0.50$
(393\,ms), with all three retention levels eligible.
(3) \textbf{A100 (prior data) — both variants add overhead.}
On A100 (unpruned 289\,ms), PAPIT scores 0.686$\times$ at $k{=}0.25$;
value\_free was not benchmarked, but even $\approx$18\,ms text overhead
is unlikely to be recovered at this baseline latency.
(4) \textbf{value\_free $\approx$ Random in latency.}
On T4 at $k{=}0.25$, value\_free (531\,ms) lags Random (515\,ms) by
only 16\,ms; on L4 the gap is ${<}$1\,ms (332\,ms vs.\ 331\,ms).
Scoring overhead is effectively eliminated, enabling prompt-aware
token selection at the cost of random pruning.

\section{Conclusion}

We have implemented and validated the full PAPIT pipeline end-to-end:
hybrid GradCAM / value-feature scoring, top-$k$ spatial-anchor pruning, and
token-level LLaVA integration.
The key technical contribution is \emph{value\_free} scoring: by registering a
lightweight hook on \texttt{v\_proj} during the already-running ViT forward pass,
we reduce the fixed scoring overhead from $\approx$130\,ms (GradCAM/cosine hybrid)
to $\approx$18\,ms (text encoding only).

End-to-end benchmarks on 100 images (5 timed runs, 3 warmup discarded) confirm
that value\_free achieves \textbf{1.65$\times$} speedup at $k{=}0.25$ on T4
(531\,ms vs.\ 876\,ms) and \textbf{1.34$\times$} on L4 (332\,ms vs.\ 445\,ms),
meeting deployment guardrails at all tested $k$ on both GPU tiers
(Figure~\ref{fig:throughput}).
In contrast, original PAPIT achieves 1.36$\times$ on T4 at $k{=}0.25$ but
\emph{fails at every retention level} on L4---a hardware-tier boundary
identified empirically: the $\approx$130\,ms overhead exceeds available token
savings once the unpruned LLM baseline drops below $\approx$750\,ms.
value\_free eliminates this boundary, achieving latency at parity with random
pruning ($<$16\,ms gap on T4 at $k{=}0.25$) while selecting patches semantically.

For accuracy, the hybrid scorer narrows the gap to random pruning throughout:
on GQA it \emph{surpasses} random at $k{=}75\%$ (70.0\% vs.\ 69.3\%);
on TextVQA, PAPIT's text-region patch recall exceeds random at $k{=}25\%$
(40.8\% vs.\ 33.9\%), confirming that the scorer preferentially retains
semantically relevant patches even without explicit OCR guidance.

Taken together, these results establish that training-free prompt-aware token
pruning is practically viable on commodity GPU tiers via value\_free scoring,
achieving up to \textbf{1.65$\times$} wall-clock speedup with negligible
accuracy degradation over random pruning.
